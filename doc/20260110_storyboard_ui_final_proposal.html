🎬 実際の作業フロー

1️⃣
GPS・企画情報を入力
左カラムで企画の目標や計画を記入
↓
2️⃣
シーンを追加してセリフを書く
中央カラムでシーン#1, #2, #3... を追加
↓
3️⃣
シーンを選択 → 画像生成タブで画像を作成
シーン#1をクリック → 右パネル「📷 画像生成」タブ → プロンプト入力 → 生成 → 採用
↓
4️⃣
動画生成タブに切り替えて動画を作成
右パネル「🎬 動画生成」タブをクリック → 採用画像が自動選択 → 動画プロンプト入力 → 生成
↓
5️⃣
次のシーンへ（繰り返し）
シーン#2を選択 → 画像生成 → 動画生成 → シーン#3... と繰り返し

💡 ポイント

  • • 画像生成は最初だけ - 画像ができたらタブを切り替え
  • • 動画生成がメイン作業 - 各シーンの動画を次々と生成
  • • シーンを切り替えても - 右パネルのタブ状態は維持される
  • • ページ遷移なし - 全ての作業が1つの画面で完結

🔧 実装計画（ステップバイステップ）

Step 1

HTMLマークアップ調整

  • • GPS幅を320pxに拡張（現在280px）
  • • 右パネル幅を440pxに拡張（現在384px）
  • • タブボタンを追加（画像生成 / 動画生成）
  • • 動画生成タブのHTMLを追加（モック状態）
⏱️ 所要時間: 30分
Step 2

タブ切り替え機能（JavaScript）

  • • タブボタンのクリックイベント
  • • タブ切り替え時のコンテンツ表示/非表示
  • • タブ状態の管理（localStorage等）
⏱️ 所要時間: 15分
Step 3

シーン選択時の連動処理

  • • シーン選択 → 両方のタブのシーン情報を更新
  • • 画像生成タブ → そのシーンの画像履歴を表示
  • • 動画生成タブ → 採用画像を自動選択して表示
⏱️ 所要時間: 20分
Step 4

動画生成API統合（Veo 3.1）

  • • 動画生成ボタンのクリックイベント
  • • Supabase Edge Function経由でVeo 3.1呼び出し
  • • 生成状況のポーリング処理
  • • 動画履歴への追加
  • • ダウンロード機能
⏱️ 所要時間: 1時間（既存のapi-test-movie.htmlから移植）
